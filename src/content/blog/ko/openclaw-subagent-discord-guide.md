## 서브에이전트의 필요성

혼자서 모든 일을 처리하는 것보다, 역할을 나눠서 협업하는 것이 훨씬 효율적이라는 건 누구나 아는 사실이에요. AI 에이전트도 마찬가지입니다.

하나의 메인 에이전트가 스펙 기획, 코드 개발, 리서치, 문서 작성까지 전부 순차적으로 처리한다면 시간이 오래 걸릴 수밖에 없어요. 하지만 서브에이전트를 활용하면 이야기가 달라집니다.

**서브에이전트의 장점:**

- **병렬 처리**: 여러 작업을 동시에 수행하여 전체 소요 시간을 대폭 단축해요.
- **역할 분담**: 각 에이전트가 특화된 역할(기획, 개발, 리서치 등)을 맡아 품질을 높여요.
- **컨텍스트 분리**: 메인 대화의 컨텍스트가 비대해지는 것을 방지하고, 각 서브에이전트가 독립적인 환경에서 작업해요.
- **비용 최적화**: 간단한 작업은 저렴한 모델에게, 복잡한 작업은 고성능 모델에게 맡길 수 있어요.

예를 들어, "새로운 기능을 개발해줘"라는 요청이 들어오면 메인 에이전트는 다음과 같이 작업을 분배할 수 있어요:

1. **리서치 에이전트**: 관련 기술 문서와 베스트 프랙티스 조사
2. **기획 에이전트**: 요구사항 정리 및 스펙 문서 작성
3. **Pi 에이전트**: 실제 코드 구현 (유일한 코딩 에이전트 경로)

이 모든 작업이 병렬로 진행되니, 마치 팀 전체가 동시에 움직이는 것과 같은 효과를 얻을 수 있어요.

## 서브에이전트 등록 및 허용

서브에이전트를 사용하려면 먼저 `openclaw config` 명령어를 통해 허용 목록에 모델을 등록해야 해요.

### 설정 파일 열기

터미널에서 다음 명령어를 실행하세요:

```bash
openclaw config
```

이 명령어는 OpenClaw의 설정 파일을 편집기로 열어줍니다.

### agents.allowlist 설정

설정 파일에서 `agents` 섹션을 찾아 `allowlist`에 사용할 모델들을 추가해 주세요:

```json
{
  "agents": {
    "allowlist": [
      "glm-4.7",
      "gpt-4.1-mini",
      "claude-3-haiku",
      "deepseek-chat"
    ]
  }
}
```

**주요 설정 항목:**

| 항목 | 설명 |
|------|------|
| `allowlist` | 서브에이전트로 사용 가능한 모델 목록이에요. |
| `defaultModel` | 서브에이전트 호출 시 기본으로 사용할 모델을 지정해요. |
| `maxConcurrent` | 동시에 실행 가능한 서브에이전트 수를 제한해요. |

### 모델별 권한 세분화

특정 모델에게만 특정 도구 사용 권한을 부여할 수도 있어요:

```json
{
  "agents": {
    "allowlist": ["glm-4.7", "gpt-4.1-mini"],
    "permissions": {
      "glm-4.7": {
        "tools": ["web_search", "fs_read"],
        "maxTokens": 4096
      },
      "gpt-4.1-mini": {
        "tools": ["fs_read", "fs_write", "execute_bash"],
        "maxTokens": 8192
      }
    }
  }
}
```

이렇게 하면 리서치용 에이전트에게는 웹 검색 권한만, 개발용 에이전트에게는 파일 시스템 접근 권한을 부여하는 식으로 세밀하게 제어할 수 있어요.

## 스킬 시스템 이해하기

OpenClaw는 스킬(Skills) 시스템을 통해 에이전트의 기능을 확장합니다. 스킬은 다음 순서로 검색됩니다:

1. **`<workspace>/skills`** - 워크스페이스 스킬 (최우선)
2. **`~/.openclaw/skills`** - 글로벌 스킬
3. **번들 스킬** - 기본 제공 스킬

이 우선순위를 활용하면 프로젝트별로 커스텀 스킬을 정의하거나, 글로벌 스킬을 오버라이드할 수 있어요.

## 다중 디스코드 봇 설정

OpenClaw의 강력한 기능 중 하나는 여러 디스코드 봇을 동시에 운영하거나, 하나의 봇에서 채널별로 다른 에이전트를 매핑하는 '멀티 페르소나' 환경을 구축할 수 있다는 점이에요.

### 방법 1: 여러 봇 토큰 관리

각각 다른 성격의 봇을 운영하고 싶다면, 설정 파일에 여러 봇 토큰을 등록할 수 있어요:

```json
{
  "discord": {
    "bots": [
      {
        "name": "개발봇",
        "token": "YOUR_DEV_BOT_TOKEN",
        "model": "gpt-4.1-mini",
        "systemPrompt": "당신은 개발 전문 AI 어시스턴트입니다."
      },
      {
        "name": "리서치봇",
        "token": "YOUR_RESEARCH_BOT_TOKEN",
        "model": "glm-4.7",
        "systemPrompt": "당신은 리서치 전문 AI 어시스턴트입니다."
      }
    ]
  }
}
```

### 방법 2: 채널별 에이전트 매핑 (agentMappings)

하나의 봇으로 여러 채널에서 다른 페르소나를 운영하고 싶다면 `agentMappings`를 활용하세요:

```json
{
  "discord": {
    "token": "YOUR_BOT_TOKEN",
    "agentMappings": {
      "1234567890": {
        "name": "코딩 도우미",
        "model": "gpt-4.1-mini",
        "systemPrompt": "친절한 코딩 멘토로서 초보자도 이해하기 쉽게 설명해 주세요.",
        "temperature": 0.7
      },
      "0987654321": {
        "name": "창작 파트너",
        "model": "claude-3-haiku",
        "systemPrompt": "창의적인 글쓰기 파트너로서 영감을 주는 답변을 해주세요.",
        "temperature": 0.9
      },
      "default": {
        "name": "범용 어시스턴트",
        "model": "glm-4.7",
        "systemPrompt": "다재다능한 AI 어시스턴트입니다."
      }
    }
  }
}
```

**agentMappings 키 설명:**

- 숫자 키(예: `"1234567890"`)는 디스코드 채널 ID예요.
- `"default"`는 매핑되지 않은 채널에서 사용할 기본 에이전트를 지정해요.

### 채널 ID 확인 방법

디스코드에서 채널 ID를 확인하려면:

1. 디스코드 설정 → 고급 → 개발자 모드 활성화
2. 원하는 채널을 우클릭 → "ID 복사" 선택

## 실전 활용: 협업 시나리오 예시

실제로 메인 에이전트가 서브에이전트들과 협업하는 시나리오를 살펴볼게요.

### 시나리오: 블로그 포스트 자동 생성

사용자가 "React 18의 새로운 기능에 대한 블로그 글을 작성해줘"라고 요청했다고 가정해 볼게요.

**1단계: 메인 에이전트의 작업 분배**

메인 에이전트는 요청을 분석하고 서브에이전트들에게 작업을 할당해요:

```
[메인 에이전트]
├── [서브에이전트 A - 리서치]
│   └── "React 18 공식 문서와 릴리즈 노트 조사"
├── [서브에이전트 B - 리서치]
│   └── "React 18 관련 커뮤니티 반응 및 사용 사례 수집"
└── [서브에이전트 C - 구조 기획]
    └── "기술 블로그 포스트 아웃라인 작성"
```

**2단계: 병렬 실행 및 결과 수집**

세 서브에이전트가 동시에 작업을 수행하고, 메인 에이전트는 결과를 기다려요:

```
서브에이전트 A 완료: React 18 핵심 기능 정리 (Concurrent Features, Suspense 개선 등)
서브에이전트 B 완료: 개발자 커뮤니티 피드백 요약
서브에이전트 C 완료: 블로그 포스트 구조 제안
```

**3단계: 메인 에이전트의 통합 및 작성**

메인 에이전트는 수집된 정보를 바탕으로 최종 블로그 포스트를 작성해요. 필요하다면 추가 서브에이전트를 호출하여 코드 예제 생성이나 이미지 검색을 맡길 수도 있어요.

### 시나리오: 코드 리뷰 자동화

```
[메인 에이전트]
├── [서브에이전트 A] → 코드 스타일 및 린팅 검사
├── [서브에이전트 B] → 보안 취약점 분석
├── [Pi 에이전트] → 성능 최적화 제안 (코딩 전문)
└── [서브에이전트 D] → 테스트 커버리지 확인
```

각 서브에이전트가 전문 분야를 담당하여 종합적인 코드 리뷰 리포트를 빠르게 생성할 수 있어요.

## 마무리

서브에이전트와 다중 디스코드 봇 설정을 활용하면, OpenClaw의 가능성은 무한히 확장돼요. 혼자서 모든 일을 처리하는 단일 에이전트의 한계를 넘어, 마치 전문가 팀을 운영하는 것처럼 다양한 작업을 효율적으로 처리할 수 있습니다.

핵심 포인트를 정리하면:

- **서브에이전트**: `agents.allowlist`에 모델을 등록하여 병렬 작업 처리
- **코딩 에이전트**: 'Pi'가 유일한 코딩 에이전트 경로
- **스킬 우선순위**: `<workspace>/skills` → `~/.openclaw/skills` → 번들 스킬
- **다중 봇**: 여러 토큰으로 각각 다른 성격의 봇 운영
- **채널 매핑**: `agentMappings`로 하나의 봇에서 멀티 페르소나 구현

이제 여러분만의 AI 팀을 구성해 보세요!
