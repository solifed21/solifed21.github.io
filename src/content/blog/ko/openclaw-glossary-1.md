## 🤖 Agent (에이전트)

**한 줄 정의:** AI가 실제로 일을 수행하는 "디지털 비서"

### 쉬운 비유
Agent는 여러분의 **개인 비서**라고 생각하면 됩니다. 단순히 "이렇게 하세요"라고 조언만 하는 게 아니라, 직접 파일을 만들고, 코드를 작성하고, 명령어를 실행하는 실행력 있는 비서예요.

### 실제 예시
- "이 폴더에 README 파일 만들어줘" → Agent가 직접 파일 생성
- "이 코드의 버그 고쳐줘" → Agent가 코드를 분석하고 수정
- "오늘 날씨 알려줘" → Agent가 웹 검색 후 답변

### 기존 AI와의 차이점
| 기존 AI 챗봇 | OpenClaw Agent |
|-------------|----------------|
| "이렇게 하세요"라고 설명 | 직접 실행 |
| 대화만 가능 | 파일/코드/명령어 조작 가능 |
| 단발성 응답 | 연속 작업 수행 |

## 🚪 Gateway (게이트웨이)

**한 줄 정의:** 모든 메시지가 드나드는 "중앙 관문"

### 쉬운 비유
Gateway는 **아파트 경비실**과 같아요. 모든 방문객(메시지)은 경비실을 거쳐야 하고, 경비원이 누가 어디로 가는지 관리합니다. WhatsApp에서 온 메시지든, Discord에서 온 메시지든 모두 Gateway를 통해 Agent에게 전달됩니다.

### 왜 필요한가요?
- **통합 관리:** 여러 채팅 앱의 메시지를 한 곳에서 처리
- **보안:** 허가된 사용자만 Agent와 대화 가능
- **안정성:** 메시지 순서와 상태를 일관되게 유지

### 작동 방식
```
WhatsApp 메시지 ─┐
Telegram 메시지 ─┼─→ Gateway ─→ Agent ─→ 응답
Discord 메시지 ──┘
```

Gateway는 컴퓨터에서 계속 실행되는 프로그램이에요. `openclaw gateway` 명령어로 시작하면, 여러분의 컴퓨터가 AI 비서의 "본부"가 됩니다.

## 📱 Channel (채널)

**한 줄 정의:** Agent와 대화할 수 있는 "소통 창구"

### 쉬운 비유
Channel은 **전화번호**와 비슷해요. 같은 사람(Agent)에게 연락하더라도 집 전화, 휴대폰, 회사 전화 등 여러 번호로 연락할 수 있죠. OpenClaw에서는 WhatsApp, Telegram, Discord 등이 각각 하나의 Channel입니다.

### 지원되는 주요 채널
- **WhatsApp:** 가장 인기 있는 채널, QR 코드로 연결
- **Telegram:** 설정이 가장 간단함
- **Discord:** 서버와 DM 모두 지원
- **Slack:** 업무용 메신저 연동
- **iMessage:** macOS에서 사용 가능

### 채널별 특징
각 채널마다 지원하는 기능이 조금씩 달라요:
- 텍스트 메시지: 모든 채널 지원 ✅
- 이미지/파일: 대부분 지원
- 음성 메모: 일부 채널만 지원
- 이모지 반응: 채널마다 다름

## 💬 Session (세션)

**한 줄 정의:** Agent와의 "대화 맥락"을 기억하는 단위

### 쉬운 비유
Session은 **대화 노트**와 같아요. 친구와 카톡할 때 이전 대화 내용을 보면서 맥락을 이해하듯이, Agent도 Session을 통해 "아까 뭘 부탁했더라?"를 기억합니다.

### 세션이 없다면?
```
나: "파이썬 코드 작성해줘"
Agent: (코드 작성)
나: "거기에 주석 추가해줘"
Agent: "어떤 코드요?" ← 맥락을 모름!
```

### 세션이 있다면?
```
나: "파이썬 코드 작성해줘"
Agent: (코드 작성)
나: "거기에 주석 추가해줘"
Agent: (방금 작성한 코드에 주석 추가) ← 맥락을 기억!
```

### 세션의 종류
- **Main Session:** 1:1 DM 대화는 하나의 메인 세션으로 합쳐짐
- **Group Session:** 그룹 채팅은 별도 세션으로 분리

## 🏠 Workspace (워크스페이스)

**한 줄 정의:** Agent가 일하는 "작업 공간"

### 쉬운 비유
Workspace는 Agent의 **사무실**이에요. 책상 위에 필요한 서류(파일)가 있고, 메모장(설정 파일)이 있고, 작업 도구(Tools)가 준비되어 있는 공간입니다.

### 워크스페이스에 있는 것들
```
~/.openclaw/workspace/
├── AGENTS.md      ← Agent의 행동 지침
├── SOUL.md        ← Agent의 성격/말투 설정
├── TOOLS.md       ← 사용 가능한 도구 목록
├── USER.md        ← 사용자 정보
└── skills/        ← 추가 기능들
```

### 왜 중요한가요?
- Agent는 이 폴더 안에서만 파일을 읽고 쓸 수 있어요
- 설정 파일을 수정하면 Agent의 행동이 바뀝니다
- 안전하게 격리된 환경에서 작업이 이루어져요

## 🔧 Tool (도구)

**한 줄 정의:** Agent가 사용할 수 있는 "기능"

### 쉬운 비유
Tool은 Agent의 **연장통**에 들어있는 도구들이에요. 망치(파일 생성), 드라이버(코드 수정), 자(웹 검색) 등 다양한 도구를 상황에 맞게 꺼내 씁니다.

### 기본 제공 도구들
- **read:** 파일 읽기
- **write:** 파일 쓰기
- **exec:** 명령어 실행
- **web_search:** 웹 검색
- **browser:** 웹 브라우저 조작

### 도구 사용 예시
```
사용자: "package.json 파일 내용 보여줘"
Agent: (read 도구 사용) → 파일 내용 출력

사용자: "npm install 실행해줘"
Agent: (exec 도구 사용) → 명령어 실행
```
